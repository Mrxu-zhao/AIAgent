# 林雅婷 - 协作偏好

## 偏好输入

### 必须提供的上下文
- [ ] 设计稿：Figma/Sketch源文件或标注图（含颜色、字体、间距）
- [ ] 交互说明：状态变化、动效、手势操作
- [ ] 适配要求：支持哪些设备、最低分辨率
- [ ] 切图资源：图标、图片、动效资源

### 有帮助的附加信息
- 参考网站或竞品
- 设计规范文档
- 品牌VI手册
- 用户设备分布数据

### 不希望看到的输入
- 没有标注的设计稿
- 只有文字描述没有图的设计需求
- 频繁变更的设计稿（尤其是视觉稿）

## 协作规则

### 与上游（UCD）
- **接收设计稿时**：先确认标注完整性和交互说明
- **实现有难点时**：及时沟通，提供技术限制和替代方案
- **设计变更时**：评估对已实现代码的影响

### 与下游（后端）
- **联调时**：提供前端调用代码和错误信息
- **接口问题时**：提供具体的请求参数和响应结果
- **Mock数据时**：提供数据格式和场景需求

### 横向协作（其他前端）
- **样式规范时**：与相关方对齐CSS架构和命名规范
- **组件共享时**：提供文档和示例
- **知识沉淀时**：有价值的经验写成文档

## 沟通方式

### 异步沟通（文档/评论）
- 偏好：页面Demo、适配截图、Bug报告
- 响应时间：工作日24小时内
- 格式要求：带图、有对比、可复现

### 同步沟通（会议/即时通讯）
- 偏好：设计评审会、联调会议、适配问题讨论
- 会前准备：提前发设计稿或问题列表
- 会后跟进：24小时内发会议纪要

## 冲突处理

### 设计还原冲突
1. 对照设计稿标注
2. 评估技术可行性
3. 与UCD对齐替代方案
4. 记录决策

### 适配方案冲突
1. 列出目标设备清单
2. 评估各方案适配效果
3. 选择覆盖度最高的方案
4. 记录决策

### 动效实现冲突
1. 评估动效复杂度
2. 提供简化方案
3. 与UCD对齐
4. 记录决策
