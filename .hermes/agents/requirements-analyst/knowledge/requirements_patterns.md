# 需求分析模式库

> 从项目中提炼的需求分析方法和最佳实践

## 模式4: 业务边界划分（微服务需求分析）

- **分类**: 架构需求
- **来源**: xzmeto_resource 微服务架构分析
- **适用场景**: 微服务架构项目需求分析
- **模式描述**:

### 业务边界检查清单

| 检查项 | 说明 | 验证方法 |
|--------|------|----------|
| 代码归属 | 服务代码是否与业务边界一致 | 检查各服务 package 路径 |
| 数据库归属 | 数据库表是否与代码服务一致 | 检查表前缀或库名归属 |
| 接口归属 | API 是否归到正确的服务 | 检查 Controller 所在服务 |
| 共享数据 | 跨服务共享数据如何处理 | 检查 Feign 调用链 |

### 常见问题

1. **假微服务**: 代码分开了但数据库共享（业务边界模糊）
2. **过度拆分**: 微服务拆得太细，服务间调用链路复杂
3. **数据不一致**: 两个服务都持有同一份数据，修改时不同步

### 需求分析要点

```
Step 1: 识别核心业务域（如 xzmeto 的"资源管理"）
Step 2: 划分边界（每个域一个服务？还是有多个？）
Step 3: 确定跨域交互（同步/异步？）
Step 4: 确定共享数据（哪些数据多个域都要用？）
Step 5: 验证边界（代码+数据库+接口三者归属一致？）
```

## 模式5: 需求分析优先级排序

- **分类**: 需求管理
- **来源**: mall4j + xzmeto 综合分析
- **适用场景**: 新项目需求分析启动阶段
- **模式描述**:

### 需求分层模型

| 层级 | 内容 | 示例 | 优先级 |
|------|------|------|--------|
| L1 核心域 | 业务核心链路 | 电商: 商品→订单→支付 | 最高 |
| L2 支撑域 | 核心域的支撑功能 | 电商: 用户/商品管理 | 高 |
| L3 扩展域 | 增值功能 | 电商: 分销/直播/抽奖 | 中 |
| L4 运营域 | 辅助运营功能 | 电商: 统计报表/客服 | 低 |

### mall4j vs xzmeto 优先级对比

| 系统 | L1 核心域 | L2 支撑域 |
|------|----------|----------|
| mall4j 电商 | 订单+支付+物流 | 用户+商品+促销 |
| xzmeto 管理 | 资源+认购+合同 | 用户+审批+支付 |

## 模式6: 多端需求矩阵

- **分类**: 多端适配
- **来源**: mall4j 四套前端 + xzmeto 两套前端
- **适用场景**: 多端配套项目
- **模式描述**:

### 需求采集矩阵

| 维度 | PC管理后台 | PC商家后台 | 门店系统 | 小程序 |
|------|------------|------------|----------|--------|
| 用户角色 | 平台运营 | 入驻商家 | 门店店员 | 消费者 |
| 核心流程 | 运营管理 | 商品上下架 | 收银核销 | 选购下单 |
| 鉴权方式 | 后台账号+密码 | 后台账号+密码 | 店员刷脸/扫码 | 微信授权 |
| 权限模型 | 菜单+按钮 | 菜单+按钮 | 简单功能 | 无权限设计 |
| 技术栈 | Vue3+TS+Element | Vue3+TS+Element | Vue3+TS+Element | uni-app |
| 离线需求 | 无 | 无 | 必须离线 | 可选离线 |

### 需求采集问题清单

```
1. 角色划分：每端的用户角色是什么？
2. 核心链路：每端的核心业务链路是什么？
3. 数据共享：哪些数据多端共享？
4. 鉴权统一：多端是否用同一套鉴权体系？
5. 权限差异：各端的权限控制粒度是否不同？
6. 技术栈：各端是否统一技术栈？
7. 离线需求：哪些端需要离线能力？
```
