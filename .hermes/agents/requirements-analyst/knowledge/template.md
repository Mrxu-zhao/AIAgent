# 需求分析师 - 文档模板

## 1. SRS 需求规格说明书模板

```markdown
# [项目名称] 需求规格说明书
版本：v1.0 | 日期：YYYY-MM-DD | 作者：[需求分析师]

## 1. 引言

### 1.1 目的
本文档定义[项目名称]的功能和非功能需求，作为开发、测试和验收的唯一依据。

### 1.2 范围
本文档涵盖[系统/模块]的全部需求。

### 1.3 定义与缩略语
| 术语 | 定义 |
|------|------|
|      |      |

## 2. 总体描述

### 2.1 产品概述
[一句话描述系统是什么]

### 2.2 用户特征
- 主要用户：[描述]
- 使用频率：[高/中/低]
- 技术水平：[高/中/低]

### 2.3 约束条件
- 技术约束：[如有]
- 业务约束：[如有]
- 时间约束：[如有]

## 3. 功能需求

### 3.1 功能点 [编号]
**描述**：[清晰描述功能]
**优先级**：[高/中/低]
**前置条件**：[什么状态下可执行]
**后置条件**：[执行后产生什么结果]
**正常流程**：
1. [步骤1]
2. [步骤2]

**异常流程**：
| 异常情况 | 处理方式 |
|----------|----------|
|          |          |

**验收标准**：
| 场景 | 输入 | 预期输出 |
|------|------|----------|
|      |      |          |

## 4. 非功能需求

### 4.1 性能
- 响应时间：≤ [X] 秒
- 并发用户：≥ [X] 人

### 4.2 安全
- [要求]

### 4.3 兼容性
- [要求]

### 4.4 可用性
- [要求]

## 5. 数据需求
[数据字典、接口规范]

## 6. 附录
[参考资料、术语表]
```

## 2. 用户故事模板

```markdown
## 用户故事：[编号]

**标题**：[作为XXX，我想要XXX，以便XXX]

**详细描述**：
- **角色**：[谁]
- **功能**：[做什么]
- **价值**：[为什么]

**验收标准**（Given-When-Then格式）：
```
Given [前置条件]
When [触发动作]
Then [预期结果]
```

**优先级**：[Must/Should/Could/Won't]
**工作量**：[?]
**备注**：[如有]
```

## 3. 需求分析检查清单

```
□ 需求来源清晰（谁提的、为什么提）
□ 业务背景已记录
□ 功能点无歧义
□ 覆盖正常流程
□ 覆盖异常流程
□ 覆盖边界条件
□ 每个功能有验收标准
□ 非功能需求已明确
□ 依赖关系已梳理
□ 优先级已评估
□ 风险点已识别
□ 干系人已确认
□ 与现有系统关系已明确
□ 术语已统一
□ 无重复需求
□ 无矛盾需求
```

## 4. 需求评审检查清单

```
□ 评审材料提前发出（≥2个工作日）
□ 业务方到场
□ 技术方到场
□ 每个功能点逐条评审
□ 异常场景已覆盖
□ 验收标准可测试
□ 优先级达成一致
□ 遗留问题已记录
□ 评审结论明确
□ 下一步行动计划确定
□ 评审纪要已发送
□ 需求文档已更新
□ 评审签字/确认已获取
```
