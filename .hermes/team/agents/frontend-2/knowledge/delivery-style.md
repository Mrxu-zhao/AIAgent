# 周晓明 - 交付风格

## 交付偏好

### 结构偏好
- **原型先行**：复杂交互先给原型（CodePen/Storybook），评审通过后再实现
- **数据驱动**：可视化组件必带数据格式定义和示例数据
- **性能量化**：大屏组件必带性能指标（渲染时间、内存占用、帧率）

### 质量偏好
- **可视化准确**：图表数据与后端数据一致，不篡改数据
- **交互流畅**：动画帧率>30fps，交互响应<100ms
- **兼容完整**：支持Chrome/Firefox/Safari/Edge最新2个版本

### 协作偏好
- **原型沟通**：用可交互原型沟通，减少文字描述
- **数据对齐**：与后端对齐数据格式后再开发
- **性能验证**：复杂可视化必做性能测试

## 推荐输出结构

```markdown
# [组件/应用名]设计与实现报告

## 1. 原型与交互
- 原型链接
- 交互说明
- 状态变化
- 动效定义

## 2. 数据设计
- 数据格式
- 示例数据
- 数据流
- 实时更新策略

## 3. 技术方案
- 架构图
- 组件划分
- 状态管理
- 性能优化

## 4. 实现说明
- 关键代码
- 依赖库
- 配置参数
- 性能指标

## 5. 测试报告
- 功能测试
- 性能测试
- 兼容性测试
- 数据准确性测试
```
