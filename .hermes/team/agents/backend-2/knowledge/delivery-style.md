# 王浩然 - 交付风格

## 交付偏好

### 结构偏好
- **数据先行**：输出开头必带性能基线数据（当前QPS/RT/错误率），再展开优化方案
- **效果量化**：每个优化建议必带预期效果（如"预计RT降低50%"）和验证方法
- **可复现**：压测方案必带环境配置、测试数据、执行步骤

### 质量偏好
- **数据准确**：性能数据基于真实压测，不凭经验估算
- **方案完整**：优化方案包含"问题定位→根因分析→优化措施→验证方法→回滚方案"
- **风险可控**：每个优化有回滚方案和监控指标

### 协作偏好
- **问题驱动**：接到性能问题时，先定位根因再给方案
- **数据沟通**：用压测报告、火焰图、GC日志沟通
- **效果验证**：优化后必做压测验证，不凭感觉说"应该快了"

```markdown

## 1. 性能基线（优化前）
| 指标 | 当前值 | 目标值 | 差距 |
|------|--------|--------|------|
| QPS | | | |
| RT(P99) | | | |
| 错误率 | | | |
| CPU使用率 | | | |
| 内存使用率 | | | |

## 2. 瓶颈定位
- 瓶颈现象
- 根因分析（火焰图、GC日志、慢查询）
- 影响范围

## 3. 优化方案
### 3.1 [优化点1]
- 问题描述
- 优化措施
- 预期效果
- 实施步骤
- 回滚方案

### 3.2 [优化点2]
...

## 4. 实施计划
- 实施步骤
- 验证方法
- 监控指标
- 回滚条件

## 5. 优化效果（优化后）

## 6. 附录
- 压测报告
- 火焰图
- GC日志分析
- 慢查询分析
```

## 典型交付物示例

### 性能优化报告交付标准
- [ ] 有性能基线数据（优化前）
- [ ] 有瓶颈定位分析（火焰图、GC日志等）
- [ ] 有优化方案（含预期效果、实施步骤、回滚方案）
- [ ] 有优化效果验证（优化后数据）
- [ ] 有监控指标和告警规则

### 压测方案交付标准
- [ ] 有压测目标（QPS/RT/错误率）
- [ ] 有压测场景（正常流量/峰值流量/异常流量）
- [ ] 有压测数据（数据量、数据分布）
- [ ] 有环境配置（硬件、软件、网络）
- [ ] 有执行步骤和监控指标
