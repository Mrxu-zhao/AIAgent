# 经验教训

> 从项目中提取的前端开发经验

## 2026-04-29 首次项目分析实战

### 经验1: 多端前端分析要先确认技术栈是否一致

- **事件**: mall4j 四套前端中，PC端 (mall4vp/vs) 用 Vue3+Element Plus，移动端 (mall4ms/uni) 用 uni-app+sard-uniapp
- **教训**: 技术栈差异大的多端项目，不能简单横向对比目录规范和 API 封装方式
- **应用**: 分析多端配套时，先按技术栈分组：
  - PC Web → Vue3 + Element Plus 组
  - 移动端 → uni-app 组
  - 再分别深入分析各组的特点

### 经验2: TypeScript 完整性是代码质量的重要指标

- **事件**: xzmeto-ui 使用完整 TypeScript（有 types/ 目录），mall4j PC端仅部分使用
- **教训**: 完整 TS 项目在类型安全、代码提示、IDE支持上明显优于部分 TS 项目
- **应用**: 评估/参考前端项目时，TS 完整性是重要加分项。新项目推荐完整 TS

### 经验3: Axios 多实例模式适用于多后端域场景

- **事件**: xzmeto-ui 有 3 个 Axios 实例 (request/requestTwo/requestThree)，分域管理不同 API
- **教训**: 多实例适用于：
  1. 多个后端服务（认证服务/业务服务/第三方服务）
  2. 不同域名/端口的后端
  3. 需要不同超时/拦截器配置的场景
- **应用**: 小型项目（单一后端）用单实例即可，不需要过度设计
