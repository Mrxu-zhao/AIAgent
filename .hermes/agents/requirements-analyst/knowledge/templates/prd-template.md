# PRD模板（需求文档）

## 1. 结论摘要（1页以内）
- **业务目标**：解决什么问题，预期收益是什么（量化）
- **范围边界**：做什么、不做什么
- **关键风险**：已知风险和待确认项（3-5条）
- **建议下一步**：优先动作和责任人

## 2. 背景与输入
- **业务背景**：为什么要做这个功能
- **用户场景**：谁在什么场景下使用
- **相关需求/文档**：链接或引用
- **约束条件**：时间、预算、技术、合规

## 3. 需求详细说明

### 3.1 功能需求
| 需求ID | 需求描述 | 优先级 | 验收标准 | 关联用户故事 |
|--------|---------|--------|----------|-------------|
| REQ-001 | | P0/P1/P2 | Given...When...Then... | US-001 |

### 3.2 非功能需求
- **性能要求**：QPS、RT、并发用户数
- **安全要求**：权限控制、数据加密、审计日志
- **兼容性要求**：浏览器、设备、系统版本
- **可用性要求**：可用性指标、故障恢复时间

## 4. 用户故事
| 故事ID | 作为... | 我想要... | 以便... | 优先级 | 验收标准 |
|--------|---------|----------|---------|--------|----------|
| US-001 | | | | P0/P1/P2 | |

## 5. 业务流程
- **正常流程**：步骤1→步骤2→步骤3...
- **异常流程**：步骤1→异常→处理...
- **边界场景**：空数据、超大数据、并发操作

## 6. 界面原型（可选）
- **页面列表**：页面1、页面2...
- **关键页面**：原型链接或截图
- **交互说明**：点击、跳转、状态变化

## 7. 数据需求
- **数据模型**：关键实体、字段、关系
- **数据规则**：校验规则、计算规则、默认值
- **数据量**：当前数据量、增长预测

## 8. 风险与限制
- **已知风险**：风险描述、影响、缓解措施
- **待确认问题**：问题描述、责任人、截止日期
- **依赖关系**：依赖谁、被谁依赖

## 9. 附录
- **术语表**：专业术语解释
- **变更日志**：版本、日期、变更内容、变更人
