# 代码审查Checklist

## 使用场景
在代码提交前或代码评审时，使用此清单检查代码质量。

## 审查维度

### 1. 代码规范
- [ ] 命名规范（类名、方法名、变量名）
- [ ] 代码格式（缩进、换行、空格）
- [ ] 注释规范（类注释、方法注释、关键逻辑注释）
- [ ] 日志规范（日志级别、日志内容）

### 2. 功能正确性
- [ ] 是否满足需求
- [ ] 边界条件处理
- [ ] 异常场景处理
- [ ] 并发安全问题

### 3. 代码质量
- [ ] 代码重复（DRY原则）
- [ ] 方法长度（不超过50行）
- [ ] 类长度（不超过500行）
- [ ] 参数数量（不超过5个）
- [ ] 嵌套层级（不超过3层）

### 4. 性能
- [ ] 是否有N+1查询
- [ ] 是否有大数据量查询
- [ ] 是否有循环内调用外部服务
- [ ] 是否有内存泄漏风险

### 5. 安全
- [ ] 是否有SQL注入风险
- [ ] 是否有XSS风险
- [ ] 是否有越权访问风险
- [ ] 是否有敏感信息泄露

### 6. 测试
- [ ] 是否有单元测试
- [ ] 测试覆盖率是否达标（>80%）
- [ ] 是否有集成测试
- [ ] 是否有异常场景测试

### 7. 文档
- [ ] 接口文档是否更新
- [ ] 配置文件是否更新
- [ ] README是否更新

## 审查结论
- [ ] 通过（无需修改）
- [ ] 有条件通过（需修改后确认）
- [ ] 不通过（需重大修改后重新审查）
