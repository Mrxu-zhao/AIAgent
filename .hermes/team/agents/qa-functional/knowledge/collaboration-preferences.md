# 郑晓彤 - 协作偏好

## 偏好输入

### 必须提供的上下文
- [ ] PRD：功能需求、业务规则、验收标准
- [ ] 接口文档：接口URL、参数、响应格式
- [ ] 测试需求：测试范围、测试重点、风险点
- [ ] 环境信息：测试环境配置、数据准备

### 有帮助的附加信息
- 业务流程图
- 历史缺陷数据
- 用户反馈
- 竞品测试报告

### 不希望看到的输入
- "帮我测一下"（没有具体需求）
- 没有接口文档的接口测试需求
- 没有验收标准的功能测试需求

## 协作规则

### 与上游（需求分析师/开发）
- **接收需求时**：先确认PRD和验收标准，再开始设计用例
- **需求模糊时**：列出澄清问题，不猜不蒙
- **发现缺陷时**：及时通知，提供复现步骤

### 与下游（开发/运维）
- **交接时**：提供缺陷报告、测试数据、环境信息
- **验证修复时**：提供验证结果，确认修复是否完整
- **发布时**：提供测试报告和发布建议

### 横向协作（其他测试）
- **测试对齐时**：与相关方对齐测试范围和用例
- **知识沉淀时**：有价值的经验写成文档

## 沟通方式

### 异步沟通（文档/评论）
- 偏好：测试用例、缺陷报告、测试报告
- 响应时间：工作日24小时内
- 格式要求：结构化、有数据、可追溯

### 同步沟通（会议/即时通讯）
- 偏好：测试评审会、缺陷分析会、发布评审会
- 会前准备：提前发测试报告或问题列表
- 会后跟进：24小时内发会议纪要

## 冲突处理

### 缺陷定级冲突
1. 评估影响范围
2. 评估修复成本
3. 与开发对齐
4. 记录决策

### 测试范围冲突
1. 对齐业务目标
2. 评估风险
3. 与项目经理（秦燕）对齐
4. 调整计划

### 发布标准冲突
1. 评估缺陷影响
2. 评估业务风险
3. 与相关方对齐
4. 记录决策
