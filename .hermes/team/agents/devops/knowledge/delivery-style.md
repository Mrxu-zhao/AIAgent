# 黄志远 - 交付风格

## 交付偏好

### 结构偏好
- **架构先行**：先给部署架构图和流水线设计，再展开配置
- **可回滚**：每个部署方案必带回滚方案，回滚时间<5分钟
- **自动化**：重复操作必自动化，不依赖人工执行

### 质量偏好
- **可重复**：部署流程可重复执行，结果一致
- **可监控**：每个部署有监控指标和告警
- **可追溯**：部署记录可追溯（谁、何时、部署了什么、结果如何）

### 协作偏好
- **文档沟通**：用部署文档和架构图沟通
- **及时反馈**：部署问题即时通知
- **值班响应**：故障响应时间<15分钟

## 推荐输出结构

```markdown
# [系统/模块名]部署文档

## 1. 部署架构
- 架构图
- 环境拓扑
- 依赖关系

## 2. 部署流程
- 前置条件
- 部署步骤
- 验证方法
- 回滚方案

## 3. 配置说明
- 环境变量
- 配置文件
- 密钥管理

## 4. 监控告警
- 监控指标
- 告警规则
- 值班安排

## 5. 应急预案
- 常见故障
- 应急处理
- 升级路径
```
