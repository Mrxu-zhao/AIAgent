# 黄志远 - 最近经验

## 2026-05

### 经验：K8s部署的"蓝绿发布"实践
- **场景**：生产环境部署时，新版本有问题，回滚时间长（10分钟+）
- **做法**：采用蓝绿发布，新版本部署到绿色环境，验证通过后切换流量，回滚只需切回蓝色
- **结果**：回滚时间从10分钟降到30秒，零停机部署
- **适用前提**：有K8s集群、有负载均衡、可接受双倍资源
- **沉淀方向**：已补充到角色层《K8s部署策略》

### 经验：告警的"分级降噪"策略
- **场景**：告警太多，值班人员麻木，重要告警被淹没
- **做法**：将告警分为P0（立即处理）/P1（4小时内）/P2（24小时内），P0告警电话通知，其他告警群通知
- **结果**：P0告警响应时间从30分钟降到5分钟，告警疲劳消除
- **适用前提**：告警数量多、有值班制度、有分级标准
- **沉淀方向**：已补充到角色层《告警治理方案》

## 2026-04

### 经验：Docker镜像的"多阶段构建"优化
- **场景**：Docker镜像体积过大（1GB+），构建时间长（10分钟+）
- **做法**：采用多阶段构建，编译阶段用完整镜像，运行阶段用精简镜像（alpine）
- **结果**：镜像体积从1GB降到100MB，构建时间从10分钟降到3分钟
- **适用前提**：使用Docker、有编译阶段、运行时依赖少
- **沉淀方向**：已补充到角色层《Docker镜像优化指南》
