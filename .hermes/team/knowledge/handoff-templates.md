# Handoff 模板

## 标准交接模板

```markdown
# Handoff

- 来源 Agent：
- 目标 Agent：
- 来源步骤：
- 目标步骤：
- 任务摘要：
- 业务目标 / 成功标准：
- 已完成内容：
- 输出产物：
- 关键决策：
- 风险与限制：
- 环境 / 数据 / 配置依赖：
- 待确认问题：
- 推荐下一步：
- 推荐执行后端：
```

## 需求 -> 架构
- 必填字段：业务目标量化指标、范围边界、核心用户故事、验收标准。
- 非功能要求：性能、安全、可用性、合规优先级必须明确。
- 边界说明：已确认边界与待确认边界分开列出，避免架构阶段重复澄清。

## 架构 -> 开发
- 必填字段：模块边界、接口契约、数据约束、异常流程、风险优先级。
- 实施约束：明确不可突破的架构边界、技术栈约束、依赖前置条件。
- 落地方式：给出实现顺序、联调顺序和首批验证点。

## 开发 -> 测试
- 必填字段：变更范围、回归点、部署分支/环境、验证方式、测试数据依赖。
- 缺陷预期：注明已知限制、暂不修复项、需要重点回归的高风险链路。
- 交付物：接口文档、示例请求、日志定位入口、开关配置。

## DBA -> 后端
- 必填字段：表设计评审意见、字段约束、索引建议、慢查询预警、容量风险。
- 数据策略：分库分表、读写分离、事务边界和归档策略要写清楚。
- 开发动作：明确必须修改的 SQL、ORM 映射、迁移脚本和回滚方案。

## UCD -> 前端
- 必填字段：设计稿链接/版本、页面状态定义、交互说明、标注规则、响应式规则。
- 组件约束：复用组件、样式 token、动效时长、禁用态和空态表现必须明确。
- 对齐方式：说明哪些视觉细节可协商，哪些属于必须还原项。

## 前端 -> 后端
- 必填字段：接口联调问题、字段映射、错误码诉求、Mock 缺口、联调阻塞项。
- 协议对齐：字段命名、枚举值、分页排序、时间格式和鉴权方式需要逐项确认。
- 验证口径：提供最小可复现请求和期望响应样例。

## 测试 -> 运维
- 必填字段：测试环境配置、性能基线、容量预估、监控诉求、发布风险等级。
- 上线关注：需要重点观察的接口、阈值、告警项和回滚触发条件。
- 交付物：压测报告、测试结论、监控看板建议、巡检清单。

## 运维 -> 开发
- 必填字段：故障报告、环境变更、配置更新、资源瓶颈、日志与告警证据。
- 定位信息：问题时间线、影响范围、复现条件、临时缓解动作必须齐全。
- 开发动作：明确需要代码修复、配置修复还是数据修复，以及验证窗口。
