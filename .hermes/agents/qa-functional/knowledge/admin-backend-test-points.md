# 管理后台常见功能测试点

> 作者：郑晓彤 | 创建时间：2026-04-29 | 适用：B端管理后台系统

## 1. 用户管理模块

### 1.1 功能清单

```markdown
□ 用户列表查询
  - 分页查询
  - 条件筛选（用户名、状态、部门、角色）
  - 排序（按创建时间、更新时间、用户名）
  
□ 新增用户
  - 基本信息录入
  - 角色分配
  - 部门分配
  - 初始密码设置
  
□ 编辑用户
  - 信息修改
  - 状态变更
  - 角色变更
  
□ 删除用户
  - 单个删除
  - 批量删除
  - 删除确认
  
□ 用户详情
  - 查看完整信息
  - 查看关联数据
  
□ 密码管理
  - 修改密码
  - 重置密码
  - 密码策略校验
```

### 1.2 测试检查点

```markdown
### 用户新增测试点

□ 用户名唯一性校验
  - 输入已存在的用户名 → 提示"用户名已存在"
  
□ 用户名格式校验
  - 空用户名 → 提示"用户名不能为空"
  - 少于3位 → 提示"用户名长度3-20位"
  - 多于20位 → 提示"用户名长度3-20位"
  - 特殊字符 → 提示"只能输入字母数字"
  
□ 密码格式校验
  - 空密码 → 提示"密码不能为空"
  - 少于6位 → 提示"密码长度6-20位"
  - 多于20位 → 提示"密码长度6-20位"
  - 纯数字/纯字母 → 建议使用混合密码（如果策略要求）
  
□ 邮箱格式校验
  - 空邮箱 → 如果必填，提示"邮箱不能为空"
  - 错误格式 → 提示"邮箱格式不正确"
  
□ 手机号格式校验
  - 错误格式 → 提示"手机号格式不正确"
  
□ 角色分配
  - 未选择角色 → 提示"请选择角色"
  - 选择多个角色 → 保存成功
  
□ 部门分配
  - 未选择部门 → 提示"请选择部门"
  - 选择子部门 → 保存成功
  
□ 重复提交
  - 快速点击"保存"按钮两次 → 只创建一条记录
```

```markdown
### 用户编辑测试点

□ 信息修改
  - 修改用户名 → 验证用户名唯一性
  - 修改邮箱 → 验证邮箱格式
  - 修改手机号 → 验证手机号格式
  
□ 状态变更
  - 启用→禁用：用户无法登录
  - 禁用→启用：用户恢复登录
  - 不能禁用自己 → 提示"不能禁用当前账号"
  
□ 角色变更
  - 移除所有角色 → 提示"至少保留一个角色"
  - 新增角色 → 保存成功
  
□ 乐观锁
  - 模拟并发修改 → 后提交的应被拒绝或覆盖
  
□ 禁用中的用户
  - 编辑禁用用户 → 可以正常编辑
```

```markdown
### 用户删除测试点

□ 单个删除
  - 点击删除 → 弹出确认框
  - 确认删除 → 删除成功
  - 取消删除 → 数据未删除
  
□ 批量删除
  - 选择多条记录 → 删除成功
  - 选择1条记录 → 删除成功
  
□ 删除限制
  - 删除有订单的用户 → 提示"该用户有关联数据"
  - 删除当前登录用户 → 提示"不能删除当前账号"
  - 删除管理员账号 → 提示"不能删除管理员"
  
□ 逻辑删除 vs 物理删除
  - 逻辑删除：数据仍存在，只是标记为已删除
  - 物理删除：数据从数据库真正删除
  - 验证两种方式的业务影响
```

## 2. 权限管理模块

### 2.1 功能清单

```markdown
□ 角色管理
  - 角色列表查询
  - 新增角色
  - 编辑角色
  - 删除角色
  - 角色详情（查看关联用户）
  
□ 权限配置
  - 菜单权限分配
  - 接口权限分配
  - 数据权限配置
  
□ 用户角色绑定
  - 给用户分配角色
  - 移除用户角色
  - 批量分配角色
```

### 2.2 测试检查点

```markdown
### 角色管理测试点

□ 新增角色
  - 角色名称唯一性校验
  - 角色编码唯一性校验
  - 描述信息可选填
  
□ 编辑角色
  - 修改角色名称 → 验证唯一性
  - 修改权限配置 → 保存成功
  
□ 删除角色
  - 删除有用户的角色 → 提示"该角色下有X个用户"
  - 删除无关联的角色 → 删除成功
  
□ 超级管理员
  - 不能删除
  - 不能修改角色名称
  - 不能移除权限
```

```markdown
### 权限配置测试点

□ 菜单权限
  - 勾选菜单 → 用户可以看到菜单
  - 取消勾选 → 用户看不到菜单
  - 勾选父级 → 默认勾选所有子级
  - 只勾选子级 → 用户只能看到子级菜单
  
□ 接口权限
  - 勾选接口权限 → 可以调用接口
  - 未勾选 → 调用返回403
  
□ 数据权限
  - 仅本人数据 → 只能看自己的数据
  - 本部门数据 → 可以看本部门数据
  - 所有数据 → 可以看全部数据
```

### 2.3 权限测试场景

```markdown
## 权限验证测试矩阵

| 用户角色 | 可访问菜单 | 可调用接口 | 可看数据范围 |
|----------|------------|------------|--------------|
| 超级管理员 | 全部 | 全部 | 全部 |
| 部门经理 | 部门管理、订单查询 | GET/POST /api/orders | 本部门 |
| 普通员工 | 我的订单、我的申请 | GET /api/my/* | 仅自己 |

## 测试步骤

1. 以超级管理员登录 → 验证可以访问全部功能
2. 以部门经理登录 → 验证可以访问部门管理、订单查询
3. 以普通员工登录 → 验证只能访问"我的订单"
4. 普通员工直接访问 /api/users → 返回403
5. 普通员工访问其他用户订单 → 返回空或403
```

## 5. 审批流程测试

### 5.1 审批流通用场景

```markdown
### 审批流状态机

[草稿] → [待提交] → [待审核] → [审核中] → [已完成]
                   ↓                    ↓
              [已撤回]              [已驳回]
                                         ↓
                                    [修改中] → [待审核]
```

### 测试检查点

```markdown
□ 流程发起
  - 填写申请表单
  - 提交前数据校验
  - 提交后状态变更
  
□ 审批操作
  - 通过审批
  - 驳回审批（需填写理由）
  - 转交他人审批
  
□ 驳回重提
  - 查看驳回原因
  - 修改申请内容
  - 重新提交
  
□ 流程查询
  - 查看我的申请
  - 查看待我审批
  - 查看审批历史
  
□ 流程监控
  - 查看流程进度
  - 查看审批节点
  - 查看审批时限
```

### 5.2 审批流测试场景

```markdown
## 场景1：正常审批流程

步骤：
  1. 申请人提交申请 → 状态=[待审核]
  2. 审批人1审核通过 → 状态=[审核中]或[已完成]
  3. 审批人2审核通过 → 状态=[已完成]

预期：
  - 每步操作都有记录
  - 状态正确流转
  - 申请人收到通知

## 场景2：驳回流程

步骤：
  1. 申请人提交申请
  2. 审批人驳回（原因：资料不全）
  3. 申请人修改资料
  4. 申请人重新提交

预期：
  - 驳回后状态=[已驳回]
  - 显示驳回原因
  - 重新提交后状态=[待审核]

## 场景3：会签流程

步骤：
  1. 申请人提交申请
  2. 需要多人同时审批
  3. 审批人A通过
  4. 审批人B通过
  5. 全部通过后完成

预期：
  - 所有人都审批后才完成
  - 显示各审批人状态

## 场景4：或签流程

步骤：
  1. 申请人提交申请
  2. 多人都可以审批
  3. 任意一人审批后完成

预期：
  - 第一人审批后流程结束
  - 第二人看到"已处理"提示

## 场景5：并发审批

步骤：
  1. 申请待审批（A和B都可以审批）
  2. A先点击"通过"
  3. B后点击"通过"

预期：
  - 只有一人成功
  - 另一人收到"已处理"提示
```

## 6. 通用测试风险点

### 6.1 高风险功能

```markdown
□ 权限相关
  - 用户权限变更
  - 角色权限变更
  - 数据权限配置
  
□ 财务相关
  - 订单金额计算
  - 退款/返款
  - 发票生成
  
□ 删除操作
  - 物理删除（数据不可恢复）
  - 级联删除
  - 批量删除
  
□ 数据导入导出
  - 批量导入
  - 大数据导出
  - 格式校验
```

### 6.2 易漏测场景

```markdown
□ 边界值
  - 最大/最小输入
  - 分页边界
  - 日期边界
  
□ 权限绕过
  - URL直接访问
  - API直接调用
  - Cookie篡改
  
□ 并发问题
  - 重复提交
  - 库存超卖
  - 数据覆盖
  
□ 兼容性
  - 浏览器兼容
  - 分辨率适配
  - 移动端适配
  
□ 网络异常
  - 网络断开
  - 请求超时
  - 服务不可用
```
