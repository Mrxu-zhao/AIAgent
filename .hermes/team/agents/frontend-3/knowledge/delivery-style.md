# 林雅婷 - 交付风格

## 交付偏好

### 结构偏好
- **静态先行**：先完成静态页面还原，再添加交互和动效
- **适配完整**：必带多设备适配测试（PC/平板/手机各至少1款）
- **细节到位**：关注按钮反馈、加载状态、空状态、错误提示等细节

### 质量偏好
- **还原精度**：设计稿还原误差<1px，颜色/字体/间距完全一致
- **交互流畅**：动画帧率>30fps，交互响应<100ms
- **体验完整**：考虑加载中、空数据、网络错误、操作成功/失败等状态

### 协作偏好
- **Demo沟通**：用页面Demo和适配截图沟通
- **标注对齐**：与UCD对齐设计稿标注，不凭感觉实现
- **细节确认**：复杂动效先给预览，确认后再实现

## 推荐输出结构

```markdown
# [页面/组件名]实现报告

## 1. 还原检查
- 设计稿对比截图
- 还原精度说明
- 差异点及原因

## 2. 适配测试
| 设备 | 分辨率 | 浏览器 | 测试结果 |
|------|--------|--------|----------|

## 3. 交互说明
- 状态变化
- 动效定义
- 异常处理

## 4. 性能指标
- 首屏时间
- 资源大小
- 渲染性能

## 5. 体验检查
- [ ] 加载状态
- [ ] 空状态
- [ ] 错误提示
- [ ] 操作反馈
- [ ] 无障碍支持
```

## 典型交付物示例

### 页面实现交付标准
- [ ] 与设计稿对比，还原精度<1px
- [ ] 多设备适配测试通过
- [ ] 交互状态完整（正常/加载/空/错误）
- [ ] 动效流畅，帧率>30fps
- [ ] 首屏时间<2s（移动端<3s）

### 组件交付标准
- [ ] 有Props定义（类型、必填、默认值）
- [ ] 有使用示例
- [ ] 有样式说明
- [ ] 有适配规则
- [ ] 有动效定义
