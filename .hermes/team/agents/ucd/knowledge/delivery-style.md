# 吴俊杰 - 交付风格

## 交付偏好

### 结构偏好
- **架构先行**：先给信息架构和流程图，再展开页面设计
- **原型驱动**：复杂交互先给可交互原型，评审通过后再出视觉稿
- **规范完整**：设计稿必带标注、交互说明、状态定义

### 质量偏好
- **一致性**：同一系统的交互模式一致，不重复造轮子
- **可扩展性**：设计时考虑未来功能扩展，避免推翻重来
- **可落地性**：设计考虑前端实现成本，不设计无法实现的交互

### 协作偏好
- **原型沟通**：用可交互原型沟通，减少文字描述
- **标注完整**：设计稿标注完整（颜色、字体、间距、动效）
- **版本管理**：设计稿版本化，变更走changelog

## 推荐输出结构

```markdown
# [模块/系统名]交互设计文档

## 1. 信息架构
- 站点地图
- 导航结构
- 页面层级

## 2. 用户流程
- 核心任务流程
- 异常流程
- 边缘场景

## 3. 页面设计
### 3.1 [页面名]
- 页面布局
- 交互说明
- 状态定义（正常/加载/空/错误）
- 动效定义

## 4. 组件规范
- 组件列表
- 使用规则
- 扩展规则

## 5. 设计Token
- 颜色
- 字体
- 间距
- 圆角
- 阴影

## 6. 附录
- 用户研究结果
- 竞品分析
- 变更日志
```
