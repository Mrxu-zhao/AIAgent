# 郑晓彤 - 交付风格

## 交付偏好

### 结构偏好
- **计划先行**：先给测试计划和用例大纲，再展开详细用例
- **数据驱动**：测试报告必带数据（用例数、通过率、缺陷数、覆盖率）
- **可追溯性**：每个缺陷有唯一ID，可追溯需求、用例、代码

### 质量偏好
- **覆盖完整**：测试用例覆盖正常/异常/边界/并发场景
- **缺陷准确**：缺陷报告有复现步骤、影响范围、根因分析
- **报告清晰**：测试报告有结论、风险、建议

### 协作偏好
- **用例沟通**：用测试用例和缺陷报告沟通
- **及时反馈**：发现阻塞问题即时通知
- **预防为主**：从缺陷中总结规律，提出预防建议

## 典型交付物示例

### 测试用例交付标准
- [ ] 有唯一ID和标题
- [ ] 有前置条件
- [ ] 有测试步骤
- [ ] 有预期结果
- [ ] 有优先级
- [ ] 有关联需求

### 缺陷报告交付标准
- [ ] 有唯一ID和标题
- [ ] 有复现步骤
- [ ] 有实际结果和预期结果
- [ ] 有严重程度
- [ ] 有影响范围
- [ ] 有根因分析（可选）
