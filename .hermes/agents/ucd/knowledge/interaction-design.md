> 来源：设计师吴俊杰 | 更新：2026-04-29

---

## 一、微交互设计四要素

| 要素 | 描述 | 示例 |
|------|------|------|
| **触发器** | 什么动作引发交互 | 点击/悬停/滚动 |
| **规则** | 交互如何响应 | 加载/变亮/展开 |
| **反馈** | 用户看到的反应 | 动画/颜色/声音 |
| **循环** | 重复/特殊状态 | 长按菜单 |

### 核心原则
- **及时性**：反馈 ≤ 100ms
- **一致性**：同类操作相同反馈
- **适度性**：避免过度动画

---

## 二、动画参数规范

| 类型 | 时长 | 缓动 | 场景 |
|------|------|------|------|
| 微反馈 | 100ms | ease-out | 按钮按下 |
| 状态切换 | 200ms | ease-out | 颜色变化 |
| 元素出现 | 250~300ms | ease-out | 弹框展开 |
| 页面过渡 | 300~400ms | ease-in-out | 页面切换 |

**推荐曲线**：
- 按钮按下：`ease-out, 100ms`
- 弹框出现：`ease-out, 250ms`
- 成功反馈：`cubic-bezier(0.68, -0.55, 0.265, 1.55), 400ms`

## 三、表单交互规范

### 即时验证
| 验证时机 | 适用场景 |
|----------|----------|
| 失焦时验证 | 大多数字段（推荐） |
| 输入时验证 | 格式校验（邮箱/手机） |
| 提交时验证 | 复杂关联校验 |

### 数量输入三方案
1. **步进器**：适合范围有限（1~99）
2. **直接输入**：适合大数量
3. **混合模式**：点击数字可编辑

## 四、移动端手势规范

| 手势 | 行为 | 场景 |
|------|------|------|
| 点击 | 触发按钮 | 所有操作 |
| 双击 | 放大图片 | 图片预览 |
| 长按 | 上下文菜单 | 商品卡长按 |
| 左滑删除 | 显示删除按钮 | 购物车 |
| 下拉刷新 | 刷新列表 | 商品列表 |

## 五、加载状态设计

| 场景 | 方案 |
|------|------|
| < 1s | 骨架屏（超过300ms显示） |
| 1~3s | 骨架屏 + 进度 |
| > 3s | 预估时间提示 |
| > 10s | 超时中断 + 重试 |

**骨架屏规范**：
- 灰色占位块模拟真实布局
- 带微妙脉冲动画（shimmer）
- 加载完成后骨架淡出

## 六、空状态组件规范

```
空状态 = 插图 + 标题 + 描述 + 操作按钮
  插图：简单清新的 SVG
  标题：说明当前状态
  描述：引导下一步
  操作：按钮或链接
```
