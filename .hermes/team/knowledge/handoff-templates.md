# Handoff 模板

## 标准交接模板

```markdown
# Handoff

- 来源 Agent：
- 目标 Agent：
- 来源步骤：
- 目标步骤：
- 任务摘要：
- 已完成内容：
- 输出产物：
- 风险与限制：
- 待确认问题：
- 推荐下一步：
- 推荐执行后端：
```

## 需求 -> 架构
- 补齐业务目标、范围边界、验收标准和未决问题。
- 明确哪些约束已经确认，哪些仍待技术判断。

## 架构 -> 开发
- 补齐模块边界、接口契约、数据约束、风险优先级。
- 明确实现顺序和不可突破的架构边界。

## 开发 -> 测试
- 补齐变更范围、验证方式、潜在回归点和环境依赖。
- 明确哪些问题属于已知限制，避免误判为缺陷。
