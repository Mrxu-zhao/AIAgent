# SQL审核Checklist

## 使用场景
在SQL提交前或代码评审时，使用此清单检查SQL的质量和性能。

## 审核维度

### 1. 语法规范
- [ ] 关键字大写（SELECT, FROM, WHERE等）
- [ ] 表名/字段名使用小写+下划线
- [ ] 有适当的缩进和换行
- [ ] 有必要的注释

### 2. 性能检查
- [ ] 是否有WHERE条件（避免全表扫描）
- [ ] WHERE条件是否有索引覆盖
- [ ] 是否使用了函数作用于索引列（导致索引失效）
- [ ] 是否使用了隐式类型转换（导致索引失效）
- [ ] 是否使用了SELECT *（只查询必要字段）
- [ ] 是否有ORDER BY（评估是否需要）
- [ ] 是否有LIMIT（避免大数据量返回）
- [ ] JOIN条件是否有索引
- [ ] 子查询是否可以改写为JOIN
- [ ] 是否使用了UNION（评估是否需要去重）

### 3. 数据安全
- [ ] 是否有SQL注入风险（使用参数化查询）
- [ ] 是否有敏感数据泄露（脱敏处理）
- [ ] 是否有越权访问（权限校验）

### 4. 事务控制
- [ ] 事务范围是否合理（不过大不过小）
- [ ] 是否有死锁风险（加锁顺序一致）
- [ ] 是否有长事务（及时提交）

### 5. 数据一致性
- [ ] 是否有并发更新冲突（乐观锁/悲观锁）
- [ ] 是否有脏读/幻读风险（隔离级别）

## 审核结论
- [ ] 通过（无需修改）
- [ ] 有条件通过（需修改后确认）
- [ ] 不通过（需重大修改后重新审核）
