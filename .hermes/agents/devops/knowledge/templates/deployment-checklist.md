# 部署Checklist

## 使用场景
在应用部署前，使用此清单检查部署准备工作的完整性。

## 检查维度

### 1. 代码检查
- [ ] 代码已合并到发布分支
- [ ] 代码已通过CI构建
- [ ] 代码已通过单元测试
- [ ] 代码已通过代码审查
- [ ] 版本号已更新
- [ ] CHANGELOG已更新

### 2. 配置检查
- [ ] 应用配置已更新
- [ ] 数据库配置已确认
- [ ] 缓存配置已确认
- [ ] 第三方服务配置已确认
- [ ] 环境变量已配置
- [ ] 密钥已配置

### 3. 数据库检查
- [ ] 数据库变更脚本已准备
- [ ] 变更脚本已测试
- [ ] 变更脚本可回滚
- [ ] 数据备份已完成

### 4. 依赖检查
- [ ] 外部服务可用
- [ ] 中间件可用
- [ ] 数据库可用
- [ ] 缓存可用
- [ ] 消息队列可用

### 5. 监控检查
- [ ] 监控指标已配置
- [ ] 告警规则已配置
- [ ] 日志收集已配置
- [ ] 值班人员已通知

### 6. 回滚准备
- [ ] 回滚方案已准备
- [ ] 回滚脚本已测试
- [ ] 旧版本包已备份
- [ ] 数据库备份已确认

- [ ] 部署成功
- [ ] 部署失败，已回滚
- [ ] 部署异常，处理中
