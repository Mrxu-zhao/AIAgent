# 孙美玲 - 协作偏好

## 偏好输入

### 必须提供的上下文
- [ ] 业务场景：核心业务流程、用户行为模式
- [ ] 性能目标：TPS、RT、并发用户数、资源使用率
- [ ] 系统架构：服务拓扑、技术栈、部署架构
- [ ] 环境信息：测试环境配置、数据量、网络环境

### 有帮助的附加信息
- 历史性能数据
- 业务增长预测
- 竞品性能指标
- 监控数据

### 不希望看到的输入
- "测一下性能"（没有具体目标）
- 没有系统架构的性能测试需求
- 没有业务场景的压测需求

## 协作规则

### 与上游（架构师/开发）
- **接收需求时**：先确认性能目标和系统架构
- **方案设计时**：与开发对齐压测场景和数据
- **发现瓶颈时**：及时通知，提供数据支撑

### 与下游（开发/运维）
- **交接时**：提供压测报告、监控数据、优化建议
- **验证优化时**：提供验证结果
- **容量规划时**：提供容量评估报告

### 横向协作（其他测试）
- **测试对齐时**：与功能测试对齐测试范围
- **知识沉淀时**：有价值的经验写成文档

## 沟通方式

### 异步沟通（文档/评论）
- 偏好：压测方案、性能报告、监控图表
- 响应时间：工作日24小时内
- 格式要求：带数据、有图表、可复现

### 同步沟通（会议/即时通讯）
- 偏好：压测评审会、瓶颈分析会、容量规划会
- 会前准备：提前发压测报告或问题列表
- 会后跟进：24小时内发会议纪要

## 冲突处理

### 性能目标冲突
1. 对齐业务目标
2. 评估技术可行性
3. 与架构师对齐
4. 记录决策

### 优化优先级冲突
1. 评估瓶颈影响
2. 评估优化成本
3. 与开发对齐
4. 记录决策

### 容量规划冲突
1. 评估业务增长
2. 评估技术约束
3. 与运维对齐
4. 记录决策
