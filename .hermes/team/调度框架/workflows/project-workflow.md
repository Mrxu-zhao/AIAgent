# 项目工作流定义

## 阶段说明

每个项目经历以下阶段，按顺序执行：

```
需求阶段 → 设计阶段 → 开发阶段 → 测试阶段 → 交付阶段
```

---

## 工作流1: 需求分析流程

```
业务方提出需求
    ↓
项目经理(秦燕) → 拆解任务
    ↓
需求分析师(吴雪梅) → 需求调研和分析
    ↓
输出: 需求文档(SRS/PRD)、业务流程图
    ↓
需求评审会议 → 通过
    ↓
进入设计阶段
```

**触发方式:**
```bash
./team-dispatch.sh requirements-analyst "分析xxx需求"
```

---

## 工作流2: 技术设计流程

```
需求评审通过
    ↓
项目经理(秦燕) → 分配设计任务
    ↓
        ┌─────────┐
        ↓         ↓
    架构师      DBA
   (张欣怡)   (周嘉诚)
        ↓         ↓
        └────┬────┘
             ↓
     输出: 技术方案、数据库设计
             ↓
       技术评审会
             ↓
        进入开发阶段
```

**触发方式:**
```bash
./team-dispatch.sh architect "设计xxx模块架构"
./team-dispatch.sh dba "设计xxx模块数据库"
```

---

## 工作流3: 开发流程

```
技术评审通过
    ↓
项目经理(秦燕) → 分配开发任务
    ↓
    ┌────┬────┐
    ↓         ↓
后端组      前端组
(3人并发)   (3人并发)
    ↓         ↓
接口对接完成
    ↓
进入测试阶段
```

**触发方式:**
```bash
# 后端开发
./team-dispatch.sh backend-1 "开发xxx模块后端"
./team-dispatch.sh backend-2 "开发xxx模块后端"
./team-dispatch.sh backend-3 "开发xxx模块后端"

# 前端开发
./team-dispatch.sh frontend-1 "开发xxx页面"
./team-dispatch.sh frontend-2 "开发xxx页面"
./team-dispatch.sh frontend-3 "开发xxx页面"
```

---

## 工作流4: 测试流程

```
开发完成，代码合并
    ↓
项目经理(秦燕) → 安排测试
    ↓
    ┌────┬────┐
    ↓         ↓
功能测试    性能测试
(郑晓彤)   (孙美玲)
    ↓         ↓
    └────┬────┘
         ↓
    测试报告
         ↓
    测试评审会
         ↓
    通过 → 进入交付阶段
    不通过 → 返回开发阶段
```

**触发方式:**
```bash
./team-dispatch.sh qa-functional "测试xxx功能"
./team-dispatch.sh qa-performance "性能测试xxx"
```

---

## 工作流5: 交付流程

```
测试通过
    ↓
项目经理(秦燕) → 安排部署
    ↓
运维(黄志远) → 部署上线
    ↓
监控和问题跟踪
    ↓
项目验收
    ↓
项目收尾和复盘
```

**触发方式:**
```bash
./team-dispatch.sh devops "部署xxx到生产环境"
```

## 并行任务模式

### 模式A: 需求+架构并行

```
         ┌──────────┐
         ↓          ↓
    需求分析师    架构师
    (吴雪梅)    (张欣怡)
         ↓          ↓
         └────┬─────┘
              ↓
         需求+架构评审
```

```
         ┌──────────┐
         ↓          ↓
       后端       前端
    (陈启明等)  (李思雨等)
              ↓
         API对接完成
```

### 模式C: 功能+性能测试并行

```
         ┌──────────┐
         ↓          ↓
      功能测试    性能测试
     (郑晓彤)    (孙美玲)
              ↓
         综合测试报告
```

## 团队沟通约定

1. **任务分配**: 秦燕统一分配任务给各Agent
2. **阻塞上报**: 任何Agent遇到问题立即上报
3. **结果归档**: 任务完成后将结果存入对应知识库
4. **评审节点**: 关键节点必须组织评审会议
