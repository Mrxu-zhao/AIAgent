# 架构评审Checklist

## 使用场景
在架构设计评审时，使用此清单检查架构方案的完整性和合理性。

## 评审维度

### 1. 功能性
- [ ] 是否满足所有业务需求
- [ ] 是否覆盖所有用户场景
- [ ] 是否处理了异常场景
- [ ] 是否考虑了边界条件

### 2. 性能
- [ ] 是否定义了性能指标（QPS/RT/并发数）
- [ ] 性能指标是否可量化
- [ ] 是否有性能瓶颈分析
- [ ] 是否有性能优化方案
- [ ] 是否有性能验证计划

### 3. 可用性
- [ ] 是否定义了可用性指标（SLA）
- [ ] 是否有冗余设计
- [ ] 是否有故障转移方案
- [ ] 是否有灾备方案
- [ ] 是否有恢复时间目标（RTO）

### 4. 安全性
- [ ] 是否有认证机制
- [ ] 是否有授权机制
- [ ] 是否有数据加密方案
- [ ] 是否有输入校验
- [ ] 是否有审计日志
- [ ] 是否考虑了OWASP Top 10

### 5. 可扩展性
- [ ] 是否支持水平扩展
- [ ] 是否支持垂直扩展
- [ ] 是否有容量规划
- [ ] 是否考虑了未来6-12个月的增长

### 6. 可维护性
- [ ] 模块划分是否清晰
- [ ] 接口定义是否稳定
- [ ] 是否有监控和告警
- [ ] 是否有日志规范
- [ ] 是否有文档

### 7. 成本
- [ ] 是否有资源成本估算
- [ ] 是否有运维成本估算
- [ ] 是否有开发成本估算
- [ ] 成本是否在预算范围内

### 8. 技术债务
- [ ] 是否识别了技术债务
- [ ] 是否有技术债务治理计划
- [ ] 是否有替代方案降低债务

## 评审结论
- [ ] 通过（无需修改）
- [ ] 有条件通过（需修改后确认）
- [ ] 不通过（需重大修改后重新评审）
