# UCD 设计踩坑记录

## 一、页面布局类

### 坑1：侧边导航折叠后图标不居中
**问题描述**：侧边栏折叠后菜单图标没有居中显示
**原因**：图标尺寸与容器不匹配
**解决方案**：
```css
/* 正确做法 */
.menu-item {
  display: flex;
  align-items: center;
  justify-content: center;
  width: 64px;
  height: 50px;
}
.menu-item .icon {
  width: 16px;
  height: 16px;
}
```
**预防措施**：在 Axure 中使用固定尺寸组件，图标统一 16x16

### 坑2：表格列宽自适应导致错位
**问题描述**：内容长度不一时表格列宽混乱
**原因**：没有设置最小列宽
**解决方案**：
- 复选框列：固定 40px
- 操作列：固定 160px
- 日期列：固定 160px
- 普通列：设置 min-width

### 坑3：弹窗内容溢出
**问题描述**：弹窗在小屏幕上超出视口
**解决方案**：
```vue
<!-- 设置最大高度 -->
<el-dialog class="custom-dialog">
  <!-- 内容区设置 overflow-y: auto -->
</el-dialog>

<style>
.custom-dialog .el-dialog {
  max-height: 90vh;
  overflow-y: auto;
}
</style>
```

## 二、表单设计类

### 坑4：必填标识不一致
**问题描述**：有的页面用红色星号，有的用「必填」文字
**规范要求**：统一使用红色星号 *
**预防**：建立表单设计规范，团队统一遵循

### 坑5：校验提示位置不对
**问题描述**：错误提示在输入框上方，影响布局
**原因**：el-form-item 位置错误
**解决方案**：确保 el-form-item 包裹 el-input

### 坑6：日期选择器时区问题
**问题描述**：选择的日期和显示的日期差一天
**原因**：时区转换问题
**解决方案**：
```javascript
// 后端统一使用 UTC 时间
// 前端提交时转换为 UTC
// 或使用 dayjs 处理时区
import dayjs from 'dayjs'
dayjs(date).format('YYYY-MM-DD')
```

### 坑7：手机号校验不支持 170 号段
**问题描述**：正则 /^1[3-9]\d{9}$/ 漏掉 170 号段
**原因**：早期正则不支持虚拟运营商号段
**解决方案**：使用完整正则
```javascript
/^1(3\d|4[5-9]|5[0-3,5-9]|6[567]|7[0-8]|8\d|9[89])\d{8}$/
// 或使用更简洁的
/^1[3-9]\d{9}$/
```

## 三、交互设计类

### 坑8：表格内编辑后数据丢失
**问题描述**：行内编辑后切换页面再返回数据未保存
**原因**：没有实时保存或状态管理问题
**解决方案**：
- 行内编辑增加确认按钮
- 或自动保存时显示保存状态
- 使用 v-model 或 watch 监听变化

### 坑9：删除操作没有确认
**问题描述**：误删数据，无法恢复
**规范要求**：所有删除操作必须二次确认
**模板**：
```
标题：确认删除
内容：确定要删除 xxx 吗？删除后数据无法恢复。
按钮：[取消] [删除]
```

### 坑10：批量操作没有限制数量
**问题描述**：用户选中几千条数据后点击操作导致系统卡死
**解决方案**：
- 选中数量上限提示（如最多100条）
- 分批操作确认
- 大数据量提示分批处理

## 四、表格设计类

### 坑11：表格没有分页导致加载慢
**问题描述**：数据量大时页面加载很慢
**解决方案**：
- 默认分页
- 支持切换每页条数
- 大数据量考虑虚拟滚动

### 坑12：排序状态不明确
**问题描述**：用户不知道当前是升序还是降序
**解决方案**：
- 排序图标清晰显示
- 升序 ↑ 降序 ↓
- 支持升序-降序-取消 三态切换

### 坑13：表格选中状态丢失
**问题描述**：翻页后选中状态消失
**原因**：分页切换时没有记录选中数据
**解决方案**：
- 记录选中 ID 列表
- 跨页选中支持
- 或明确告知用户选中状态

## 五、权限设计类

### 坑14：页面权限没有控制
**问题描述**：用户通过 URL 直接访问无权限页面
**解决方案**：
- 前端路由守卫检查权限
- 后端接口也需要权限验证
- 无权限时显示友好提示

### 坑15：按钮权限没有统一管理
**问题描述**：有的页面隐藏按钮，有的禁用
**规范要求**：无权限按钮统一隐藏，不显示禁用态

## 六、响应式类

### 坑16：表格在小屏幕上无法滚动
**问题描述**：表格宽度超出屏幕时无法横向滚动
**解决方案**：
```css
.table-container {
  overflow-x: auto;
  min-width: 100%;
}
```

### 坑17：弹窗在小屏幕上超出边界
**问题描述**：移动端弹窗太宽超出屏幕
**解决方案**：
```css
@media screen and (max-width: 768px) {
  .el-dialog {
    width: 90% !important;
    margin: 10px auto;
  }
}
```

## 七、数据可视化类

### 坑18：图表数据为0时显示异常
**问题描述**：数据为0时图表显示异常
**解决方案**：
- 空数据显示空状态
- 0 值显示为 0，不要省略
- 使用 null 表示无数据，0 表示零值

### 坑19：饼图超过5项没有处理
**问题描述**：饼图扇区太多看不清
**解决方案**：
```javascript
// 只显示前5项，剩余合并为"其他"
data: data.length > 5 
  ? [...top5, { name: '其他', value: otherSum }]
  : data
```

## 八、Axure原型类

### 坑20：原型和实际实现差距大
**问题描述**：原型高保真但开发实现后差别很大
**解决方案**：
- 使用 Element Plus 官方组件样式
- 标注详细的设计规范
- 评审时拉上开发一起

### 坑21：交互状态没有全部覆盖
**问题描述**：只有默认状态，没有 hover/disabled 等
**解决方案**：
- 组件库包含所有状态
- 原型评审检查清单覆盖所有状态
- 标注说明未实现的状态

### 坑22：Axure 文件命名混乱
**问题描述**：页面命名不规范导致沟通困难
**规范要求**：
```
命名格式：[模块]-[页面类型]-[序号]
示例：用户-列表-001，用户-新增-001
```

## 九、经验总结

### 核心原则
1. **一致性优先**：相同功能保持一致
2. **异常处理**：考虑所有边界情况
3. **用户反馈**：每个操作都要有反馈
4. **渐进增强**：先满足核心功能，再优化体验
5. **团队协作**：规范文档化，评审流程化

### 评审检查清单
- [ ] 页面加载状态（loading、空、错误）
- [ ] 表单校验（必填、格式、长度）
- [ ] 按钮状态（默认、hover、disabled、loading）
- [ ] 删除操作二次确认
- [ ] 批量操作数量限制
- [ ] 分页数据一致性
- [ ] 权限控制
- [ ] 响应式适配
- [ ] 移动端手势支持
- [ ] 无障碍支持
