# 前端性能优化Checklist

## 使用场景
在页面/组件开发完成或性能优化时，使用此清单检查前端性能。

## 检查维度

### 1. 加载性能
- [ ] 首屏时间 < 2s（PC）/ < 3s（移动端）
- [ ] 关键资源预加载（preload/prefetch）
- [ ] 图片懒加载
- [ ] 非关键资源延迟加载
- [ ] 资源压缩（Gzip/Brotli）
- [ ] 缓存策略（Cache-Control/ETag）

### 2. 渲染性能
- [ ] 避免强制同步布局（Forced Synchronous Layout）
- [ ] 使用transform和opacity做动画
- [ ] 避免大量DOM操作
- [ ] 使用虚拟滚动（大数据量列表）
- [ ] 组件按需渲染

### 3. 资源优化
- [ ] 图片压缩（WebP/AVIF）
- [ ] 图标使用SVG/IconFont
- [ ] 字体按需加载（font-display: swap）
- [ ] JS代码分割（Code Splitting）
- [ ] CSS提取（Extract CSS）

### 4. 网络优化
- [ ] HTTP/2或HTTP/3
- [ ] 资源合并（CSS/JS）
- [ ] CDN加速
- [ ] DNS预解析（dns-prefetch）

### 5. 内存优化
- [ ] 避免内存泄漏（事件监听、定时器）
- [ ] 大数据量分页加载
- [ ] 组件销毁时清理资源

### 6. 可访问性
- [ ] 图片有alt属性
- [ ] 表单有关联label
- [ ] 支持键盘操作
- [ ] 颜色对比度达标
