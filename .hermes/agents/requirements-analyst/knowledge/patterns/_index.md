> 从项目中提炼的常见需求模式和最佳实践

## 已积累模式

### 模式1: 微服务 vs 单体模块化 架构选型

- **分类**: 架构需求
- **来源**: xzmeto_resource (微服务) vs mall4j-bbc (单体模块化) 对比分析
- **适用场景**: 新项目架构选型阶段
- **模式描述**:

| 维度 | 微服务架构 | 单体模块化 |
|------|-----------|-----------|
| 团队规模 | 5人以上多团队 | 1-5人小团队 |
| 扩展需求 | 按模块独立扩展 | 统一扩展 |
| 部署复杂度 | 高（需CI/CD/K8s） | 低（单项目部署） |
| 事务控制 | 分布式事务 | 本地事务 |
| 运维成本 | 高 | 低 |
| 推荐技术 | Nacos/Gateway/Spring Cloud | Spring Boot + Maven多模块 |

- **选型建议**: 
  - 业务模块 ≥ 5个独立域 → 微服务
  - 业务模块 < 5个 或 小团队 → 单体模块化
  - 过渡方案: 先单体模块化，后期按需拆分为微服务

### 模式2: 多端前端配套 需求采集

- **分类**: 多端适配需求
- **来源**: mall4j 四套前端 (管理后台+商家后台+门店系统+小程序) 对比分析
- **适用场景**: 多端配套项目需求分析
- **模式描述**:

| 端类型 | 典型角色 | 核心功能 | 技术选型倾向 |
|--------|----------|----------|-------------|
| 管理后台 | 平台运营 | 商品管理、订单管理、用户管理、数据统计 | PC Web (Vue3+Element Plus) |
| 商家后台 | 入驻商家 | 商品上下架、订单处理、数据分析 | PC Web (Vue3+Element Plus) |
| 门店系统 | 线下店员 | 收银、库存、会员识别 | PC Web 或 Pad (Vue3+Element Plus) |
| 小程序/H5 | 消费者 | 选购、下单、支付、会员中心 | uni-app 或 原生小程序 |

- **需求采集要点**:
  1. 各端角色和核心功能不同，需求分开采集
  2. 同一后端 API 需兼容多端，数据结构要预留扩展
  3. 鉴权机制要统一（JWT Token）但权限控制要分端设计
  4. 移动端注意响应式和性能优化

### 模式3: 多租户系统 需求隔离策略

- **分类**: 权限与数据隔离
- **来源**: xzmeto_resource 多租户设计
- **适用场景**: Saas 类管理系统
- **模式描述**:

| 隔离策略 | 实现方式 | 适用场景 | 复杂度 |
|----------|----------|----------|--------|
| 字段隔离 | 所有表加 tenant_id | 低并发、多租户数据量相近 | 低 |
| 库隔离 | 每个租户独立数据库 | 高并发、数据量差异大 | 高 |
| 表隔离 | 每个租户独立表 | 租户极少（<10） | 中 |

- **需求采集要点**:
  1. 确认租户是强隔离还是弱隔离
  2. 超级管理员是否需要跨租户数据查看
  3. 租户注册/审核流程
  4. 租户到期、暂停、删除的数据处理
