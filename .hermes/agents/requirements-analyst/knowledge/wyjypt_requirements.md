## 一、项目概述

### 1.1 项目背景

徐州地铁物业经营有限公司拟建设经营数据平台，通过整合内部资源、优化管理流程、提升信息共享能力，显著提高地铁物业经营的内部管理效率。

### 1.2 项目目标

构建以**经营数据报表为核心**的一体化管理平台：
- 整合物业管理、商业管理、采购管理三大业务板块
- 实现各板块经营指标的实时反馈、统一归集与综合分析
- 形成标准化、可量化的流程绩效指标体系
- 统一业务数据口径与管理规范
- 输出完整的物业经营统计分析报表

### 1.3 项目范围

| 范围项 | 内容 |
|--------|------|
| 系统规划 | 建设目标、功能需求、性能指标 |
| 系统设计 | 数据库设计、架构设计、界面设计、模块开发 |
| 系统测试 | 功能测试、性能测试 |
| 实施上线 | 数据迁移、用户培训、系统部署 |
| 持续运维 | 系统监控、故障排查、性能优化、安全防护 |

### 1.4 与 xzmeto_resource 的关系

| 维度 | xzmeto_resource | 物业经营一体化平台 |
|------|----------------|-------------------|
| 定位 | 资源/合同/土地项目管理 | 经营数据报表+业务管理 |
| 架构 | 微服务 (Spring Cloud Alibaba) | 待定（建议参考 xzmeto） |

| 核心功能 | 资源管理+合同管理+审批流 | 经营数据采集+报表分析+集采商城 |

## 三、详细需求分析

### 3.1 物业管理

#### 3.1.1 物业管理

**业务场景:** 管理控制中心、资源开发项目、1/2/3/6号线场段、建设集团、人民广场项目、力山项目、南京安防科技、南京S3号线等物业项目。

**核心功能:**

| 功能 | 说明 | 数据来源 |
|------|------|----------|
| 服务面积统计 | 服务面积、收费标准、缴费周期 | 手工+合同系统接口 |
| 合同管理 | 从合同系统同步后认领关联物业资源和绑定商户 | 合同系统接口 |
| 物业费待缴 | 生成物业费待缴费记录，定期提醒未缴费用 | 系统生成 |
| 通知推送 | 手动触发短信催缴通知 | 系统生成 |
| 缴费处理 | 租户缴费后确认核算营收 | 手工确认 |

**业务流程:**
```
项目信息录入 → 合同系统同步 → 认领合同 → 生成缴费明细 → 催缴通知 → 缴费确认 → 营收核算
```

**报表及数据统计:**
- 服务台账（铺位/面积/品牌/合同信息）
- 收缴台账（应收/实收/欠费/收费率）
- 月租金台账、月物业费台账
- 项目收入核算报表、项目成本控制报表
- 项目盈亏计算报表

**参考复用 xzmeto_resource:**
> xzmeto_resource 的 `resource_info` + `resource_lease` + `payment_record` 模块可复用：
> - 资源信息统一管理 → 物业项目信息
> - 租赁管理 → 物业费管理
> - 缴费记录 → 收缴台账

#### 3.1.2 停车场经营

**业务场景:** 地铁大厦停车场、杏山子P+R停车场等7个停车场 + 充电桩经营。

**核心功能:**

| 功能 | 说明 | 数据来源 |
|------|------|----------|
| 停车数量统计 | 月使用频次、车辆数 | 手工导入 |
| 收入核算 | 停车费收入 | 手工导入 |
| 充电桩管理 | 每日充电订单数/订单总额/耗电量 | 手工填报 |
| 成本管理 | 设备维护、电费、人工成本 | 手工录入 |
| 盈亏计算 | 收入-成本 | 系统计算 |

**报表:**
- 停车和充电统计台账
- 市场化总收入/支出/利润
- 充电桩收入/支出明细

#### 3.1.3 光伏发电

**业务场景:** 2号线新区东、2号线新台子河、3号线后蟠桃光伏发电站。

**核心功能:**

| 功能 | 说明 | 数据来源 |
|------|------|----------|
| 发电量统计 | 日/月/年发电量 | 接口+手工 |
| 运维记录 | 设备检查、故障维修 | 手工录入 |
| 收入核算 | 系统根据发电量+电价计算 | 系统计算 |
| 成本管理 | 建设成本、运维成本、设备采购成本 | 手工录入 |
| 盈亏计算 | 系统自动计算 | 系统计算 |

**参考复用 xzmeto_resource:**
> xzmeto_resource 的 `t_land_project` + 月度计划模式可复用：
> - 项目维度管理 → 光伏电站管理
> - 月度计划/里程碑 → 运维计划

**报表:**
- 发电量统计台账（按站点、日/月度）
- 运维记录报表
- 抄表计费报表（用电公司/表号/电量/单价/金额）
- 项目收入/成本/盈亏报表

#### 3.1.4 食堂经营

**业务场景:** 地铁大厦食堂、1/2/3/6号线食堂、人保食堂、后蟠桃加工中心、外卖配餐、线上商城。

**核心功能:**

| 功能 | 说明 | 数据来源 |
|------|------|----------|

| 成本管理 | 食材采购成本、人工成本、设备维护成本 | 手工录入（需审批流） |
| 盈亏计算 | 系统自动计算食堂盈亏 | 系统计算 |

**报表:**
- 项目收入核算报表（按食堂/时间维度）
- 项目成本控制报表
- 项目盈亏计算报表
- 食材采购价格波动分析

**参考复用 xzmeto_resource:**
> 科脉系统对接模式参考 xzmeto_resource 支付模块 `pay_trade_order` 流水模式

#### 3.1.5 会议中心管理

**业务场景:** 地铁大厦7楼会议中心。

**核心功能:**
- 会议接待管理（次数/时间/人数，手工填报需审批流）
- 收入核算（按会议接待收费标准统计）
- 成本管理（人工/设备维护/场地布置）
- 盈亏计算

**报表:**
- 接待会议次数统计台账
- 项目收入/成本/盈亏报表

#### 3.1.6 其他物业管理模块

| 模块 | 核心功能 | 备注 |
|------|----------|------|
| 后期托管项目 | 项目信息、收入、成本、盈亏 | 零星项目统计 |
| 零星土地租赁 | 面积管理、总收入、成本、盈亏 | 1/2/3/6号线沿线 |
| 地铁物流 | 月运输票数、运载重量、收入、成本 | 与顺丰合作项目 |

### 3.2 商业管理

#### 3.2.1 人民广场商业管理

**核心功能:**

| 功能 | 说明 | 数据来源 |
|------|------|----------|
| 资源维护 | 商铺资源基础信息、面积统计、空置/已租/已锁定状态 | 手工维护 |
| 租赁管理 | 商铺绑定、合同签订（支持免租期） | 手工+合同系统 |
| 客户维护 | 客户信息管理、租赁管理 | 手工 |
| 缴费管理 | 租金、物业费、水电费催缴 | 系统+手工 |
| 合同认领 | 项目负责人认领合同，完成客户与资源绑定 | 手工 |
| 收支管理 | 年度收款汇总统计 | 系统 |

**业务流程:**
```
资源维护 → 合同签订 → 认领合同 → 客户绑定资源 → 缴费管理 → 催缴通知 → 到账确认
```

> - 租赁合同 → 双方合同
> - 租金催缴 → 缴费记录+预警

**报表:**
- 服务台账（铺位号/面积/品牌/合同日期/租金单价）
- 收缴台账、月租金台账、月物业费台账

### 3.3 采购管理

#### 3.3.1 采购业务板块

**核心功能:**

| 功能 | 说明 | 数据来源 |
|------|------|----------|
| 生鲜配送业务 | 第三方系统获取订单流水和营收记录 | 第三方系统接口 |

**报表:**

- 项目成本控制报表
- 项目盈亏计算报表

#### 3.3.2 内部集采系统（重点模块）

**定位:** 功能类似京东慧采，面向企业客户的商城集采系统。

**业务范围:** 大宗客户/节日礼品采买、内部低值易耗品、办公用品等。

**三端架构:**

| 模块 | 角色 | 核心功能 |
|------|------|----------|
| 线上商城 | 内外部客户 | 商品浏览、购物车、收货地址、下单、订单进度、电子发票申请 |
| 供应商门户 | 供应商 | 商品列表维护、价格报送、图片/详情更新、订单处理、物流更新 |
| 管理平台 | 物业公司管理员 | 商品审核、供应商管理、订单查看、开票申请、物流跟踪、利润统计 |

**建设规划:**

| 阶段 | 目标 | 说明 |
|------|------|------|
| 短期规划 | 内部采购 | 分子公司提需求→物业下单→供应商配送→不涉及线上付款 |
| 长期规划 | 外部扩展 | 面向内外部企业客户，对接自建物流仓储 |

**核心业务流程:**
```
供应商报价 → 系统自动比价 → 最低价上架 → 商品审核
分子公司提交需求 → 物业采购专员下单 → 二级审核 → 订单拆分
供应商接单 → 物流信息维护 → 订单完成
按时间汇总 → 采购成本/销售额/毛利/人工成本/净利润
```

**参考复用 xzmeto_resource:**
> xzmeto_resource 的商城模式 + 商品管理可参考：
> - 商品池/品类/规格维护 → 参考现有商品管理模式
> - 供应商管理 → 参考 xzmeto 的合作伙伴管理

**参考复用 mall4j-bbc:**
> 内部集采系统可参考 mall4j-bbc 电商模块：
> - 商品管理（product/sku/attr）→ 集采商品池
> - 订单管理 → 集采订单处理
> - 促销管理 → 比价逻辑

## 六、非功能性需求

### 6.1 性能需求

| 指标 | 要求 |
|------|------|
| 报表查询响应时间 | ≤ 3秒（历史数据） |
| 大屏刷新频率 | 实时/5分钟/自定义 |

| 数据同步延迟 | ≤ 1小时（外部系统） |

- 用户权限同步（对接集团主数据系统）

- 操作日志记录

### 6.3 数据质量需求

- 数据完整性校验
- 异常数据自动预警
- 数据修正功能

## 七、实施建议

### 7.1 系统架构选型

| 方案 | 适用场景 | 说明 |
|------|----------|------|
| 微服务架构（参考 xzmeto） | 长期规划、多团队 | Nacos + Gateway + OAuth2 |
| 单体模块化（参考 mall4j） | 快速上线、小团队 | Spring Boot + Maven多模块 |

**建议:** 采用微服务架构，复用 xzmeto_resource 的技术体系。

### 7.2 实施优先级

| 优先级 | 模块 | 说明 |
|--------|------|------|
| P0 | 物业管理核心（物业费+停车场+光伏）+ 大屏统计 + 业务分析 | 核心经营数据 |
| P0 | 内部集采系统（短期规划） | 新业务增长点 |
| P1 | 商业管理（商超+联合经营）+ 食堂经营 | 商业运营 |
| P2 | 会议中心/托管项目/零星土地/地铁物流/智能物管 | 辅助业务 |

### 7.3 数据对接优先级

| 优先级 | 系统 | 说明 |
|--------|------|------|
| P0 | 合同系统 | 物业管理基础数据 |
| P0 | 科脉商管系统 | 商超经营数据 |
| P1 | E餐通系统 | 食堂消费数据 |
| P1 | 光伏发电系统 | 发电量数据 |
| P2 | 京东地铁商城 | 商城数据 |
| P2 | 集团主数据系统 | 用户权限同步 |
| 待建 | 内部集采系统接口 | 自建系统对接 |
